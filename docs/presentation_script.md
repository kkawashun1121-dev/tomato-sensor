## スライド1：導入・何をしたのか（約40秒）

こんにちは、河村です。私は前期、トマトの栽培データを自動で記録・可視化するシステムを自分で作りました。

きっかけは、トマトを育てるときに「土がどれくらい乾いているか」「いつ水をやればいいか」が感覚に頼りがちだ、という点です。これをデータで見えるようにしたいと思いました。

具体的には、土に挿したセンサーで土壌水分を測り、その値を無線でサーバーに送り、パソコンやスマホのブラウザでグラフとして見られるようにしました。あわせて、天気や気温・日照、水やりや収穫といった記録も一緒に管理できるようにしています。

---

## スライド2：システムの全体像（約50秒）

システムは大きく3つの部分に分かれています。

1つ目は「測る部分」。ESP32というマイコンに、静電容量式の土壌水分センサーを3本つないでいます。

2つ目は「ためる部分」。送られてきたデータを、サーバー側のデータベースに保存します。

3つ目は「見せる部分」。ブラウザでグラフや数値として表示します。

加えて、外部の天気APIから気温・湿度・日照時間を自動で取得し、同じデータベースにためています。これで「水分」と「天気」を突き合わせて見られるようにしました。

なお発表の直前に、測定の方法を見直しました。最初は1株に1本のセンサーを挿しっぱなしにしていましたが、精度を上げるために「1つの株を3本のセンサーで測り、その平均をその株の値にする」方式に変更しました。

## スライド3：使った技術（言語・フレームワーク）（約60秒）

使った技術を、3つの部分ごとに紹介します。

まず「測る部分」のマイコンは、ESP32を Arduino の枠組み、つまりC++で書いています。センサーの誤接続で一度マイコンを壊してしまったので、今は保護抵抗を入れて安全に配線しています。

次に「サーバー部分」は、Python の FastAPI というフレームワークを使い、データベースは PostgreSQL です。これらは Docker でまとめて動かしています。データベースの操作には SQLAlchemy を使っています。

最後に「画面部分」は、React と Vite で作り、グラフの描画には Chart.js を使っています。

天気の取得には Open-Meteo という無料の天気APIを使い、コードの管理は Git と GitHub で行っています。

このように、組み込み・サーバー・フロントエンドという、種類の違う3つの領域を1つにつなげたのが、この取り組みの特徴です。

## スライド4：通信のしくみ（約60秒）

データがどう流れているかを説明します。

まず、ESP32が3本のセンサーの値を読み取ります。そして、WiFi 経由で、サーバーに向けて HTTP の POST リクエストを送ります。中身は JSON という形式で、「どのセンサーが・いくつだったか」を入れています。電池で動かすため、送信が終わるとディープスリープという省電力状態に入り、決まった時間だけ眠ってまた起きる、という動きをしています。

サーバー側の FastAPI は、そのリクエストを受け取り、データベースに保存します。

一方ブラウザは、サーバーに対して HTTP の GET リクエストを送り、保存されたデータを受け取って、Chart.js でグラフにします。

天気データは、別の Python スクリプトが定期的に天気APIへ取りに行き、同じくサーバーに POST して保存します。

つまり、機器も画面も天気も、すべて「REST API と JSON」という共通の方法でやり取りしています。違うものを同じルールでつなぐことで、後から部品を足しやすい作りになっています。

## スライド5：この後の機械学習の進め方（約60秒）

ここからは、ためたデータを「予測」に使う、機械学習のステップに進みます。

目標は、「明日の土壌水分がどれくらいになるか」、そして最終的に「次にいつ水をやるべきか」を予測することです。

入力に使うデータは、これまでためてきた、土壌水分の平均値・気温・湿度・日照時間・前回の水やりからの経過時間などです。出力は、翌日の土壌水分の値です。

手法は、いきなり複雑なものは使わず、段階的に進めます。
まずは線形回帰や、決定木・ランダムフォレストといった、結果の理由が説明しやすい回帰モデルから始めます。前日の値を特徴量として持たせる「ラグ特徴量」を作るのがポイントです。データが増えてきたら、LightGBM のような勾配ブースティングや、時系列に強い LSTM も試したいと考えています。

ライブラリは Python の scikit-learn と pandas を使います。精度の評価は、データを時間の前後で分けて、過去で学習し未来を当てる形にし、誤差は MAE や RMSE という指標で測ります。

当面の一番の課題は、データの量です。機械学習には十分なデータが必要なので、まずは1日に数回の測定を続けて、データをためることから始めます。

まとめます。前期は、トマトの栽培データを「測る・ためる・見せる」仕組みを、マイコン・サーバー・ブラウザを通して一通り作り上げました。途中、センサーの故障や通信の設定など多くの失敗もありましたが、その都度原因を調べて乗り越えました。
