# 前期取り組み（約4分）

## 3. 研究の目的・背景（40秒）

トマトを育てるとき、「いつ水をやるか」「どう育てれば甘く、たくさん採れるか」は経験や勘に頼りがちです。そこで、栽培にまつわるデータを自動で集めて見える化し、最終的には機械学習で「糖度と収穫量を高める条件」を見つけたい、と考えました。その第一歩として、前期はデータを正確に集める仕組みを作りました。

---

## 4. システムの全体像（35秒）

システムは「測る・ためる・見せる」の3つの役割でできています。測るのはマイコンとセンサー、ためるのはサーバーのデータベース、見せるのはブラウザの画面です。土壌水分は、1つの株を3本のセンサーで測り、その平均をその株の値とする方式にして、精度を高めています。

---

## 5. ハードウェア構成 〔写真〕（30秒）

〔ESP32 とセンサーの実機写真〕
測る部分の中心は、ESP32 というWiFi付きの小型マイコンです。これに、静電容量式の土壌水分センサーを3本つないでいます。土に挿すだけで水分を電気的に測れるセンサーです。

---


## 7. 技術スタック（30秒）

使った技術です。マイコンは ESP32 を Arduino、つまり C++ で書いています。サーバーは Python の FastAPI とデータベースの PostgreSQL を Docker でまとめて動かし、画面は React と Chart.js で作りました。天気は Open-Meteo という無料APIから取得し、コードは Git と GitHub で管理しています。組み込み・サーバー・画面という3つの領域を一つにつなげたのが特徴です。

---


## 9. ダッシュボード画面 〔写真〕（35秒）

〔ダッシュボード画面のスクリーンショット〕
これが実際の画面です。3本のセンサーの最新の水分量や、その平均、天気や気温、日照時間が一目で分かります。さらに、株ごとの水分の推移グラフや、水やり・収穫・糖度の履歴も見られます。

---

## 10. 設置・運用の様子 〔写真〕（25秒）

〔トマトの株とセンサー設置、運用中の写真〕
実際にトマトの株にセンサーを挿して運用している様子です。開花日や実のサイズ、糖度、写真も記録でき、開花からの累積日照時間も自動で計算しています。

---

## 11. 直面した課題と乗り越え方（35秒）

作る中では多くの課題がありました。スマホからつながらない問題は、WiFiのファイアウォール設定やネットワークの種別、サーバー側のCORS設定を見直して解決しました。iPhoneのHEIC形式の写真がアップロードできない問題は、サーバー側で自動的にJPEGに変換するようにして対応しました。また、天気データがパソコンの稼働時間に左右されて抜けてしまう問題は、1日1回まとめて取得し、抜けた日もさかのぼって補完する方式に改良しました。一つずつ原因を調べて乗り越えたことが、前期の大きな学びです。

---

# 後期・今後の取り組み（約6分）

## 12. （扉）後期・今後の取り組み（10秒）

ここからは後半、ためたデータをどう活かしていくか、後期と今後の取り組みです。

---

## 13. 後期の目標（45秒）

後期の目標は、集めたデータを使って「糖度と収穫量を高める育て方の条件」を、機械学習で見つけることです。前期で「測る・ためる・見せる」の基盤ができたので、後期はそこに「予測・分析する」を加えます。具体的には、糖度や収穫量を予測するモデルを作り、どんな条件のときに高くなるのかを明らかにしたいと考えています。

---

## 14. なぜ機械学習か（トレードオフ）（50秒）

なぜ機械学習を使うのか。理由は、トマトには「水を絞ると糖度は上がるが、収穫量や実のサイズは下がる」という相反する性質があるからです。糖度と収穫量は単純には両立せず、人が表計算で一つずつ見ても、複数の要因が絡み合った最適なバランスは見つけにくいです。土壌水分・気温・日照・水やりなど複数のデータをまとめて学習させることで、両方のバランスが良い条件を探せる——ここに機械学習を使う意味があります。

---

## 15. 使うデータ（45秒）

使うデータは、前期で集めてきたものです。出力、つまり予測したい「正解」は、糖度と収穫量。これはすでに記録できています。入力は、土壌水分の平均値、気温、湿度、日照時間、前回の水やりからの経過時間、開花からの累積日照などです。正解も入力もすでに手元で取れているので、あとはデータ量を増やしながらモデルを作っていく段階です。

---

## 16. 正解データが少ないことへの対策（50秒）

ここで一つ大事な現実があります。糖度と収穫量の「正解」は、1回の収穫につき1件しか得られません。1シーズンで集まる正解データは数十件、つまり100件未満になる可能性が高いです。

そのため、後期は次の3つの方針で進めます。

第一に、複雑すぎないモデルを軸にします。深層学習は、データが100件未満の規模だと過学習で逆に精度が落ちるためです。Ridge・Lasso、ランダムフォレスト、LightGBM、ガウス過程回帰のように、少ないデータでも安定する手法を選びます。

第二に、栽培の知識を活かした特徴量を作ります。たとえば「開花から収穫までの累積日照」「水ストレスがかかった日数」など、人の理解に近い変数を入力に加えることで、少ないデータでも本質的な関係を学びやすくします。

第三に、データを最大限活用するため、時系列を守った Leave-One-Out 交差検証を使います。未来から1件ずつテスト用に取り出して残りで学習することを繰り返し、限られたデータでも公平に精度を測ります。

---

## 17. 極端にデータが少ない場合の進め方（45秒）

データがほんの数件〜数十件のように極端に少ない場合は、手法を選ぶ以前に進め方そのものを切り替えます。フェーズに分けて役割を変えるのがポイントです。

シーズン初期、まだデータが極端に少ない Phase 1 では、無理に予測モデルを作りません。ガウス過程回帰の「予測の不確かさ」を使ったベイズ最適化で、「次に試すべき栽培条件」をシステムに提案させます。あわせて、相関分析やクラスタリング、可視化を中心とした探索的分析で「どの要因が効きそうか」の傾向を掴みます。

データが蓄積してきた Phase 2 では、Ridge・Lasso・ランダムフォレスト・LightGBM・ガウス過程回帰を段階的に試して、本格的な予測モデルを作ります。評価は時系列の Leave-One-Out 交差検証で、MAE と RMSE に加えて GPR の不確かさが実態と合っているかも検証します。

1シーズンで足りなければ、複数シーズンを連結してデータを増やす方針も取ります。

---

## 18. 機械学習の手法①：なぜこの手法か（60秒）

手法は、いきなり複雑なものではなく、理由を持って段階的に選びます。

まず線形回帰や決定木から始めます。理由は、データ量がまだ少ない段階でも扱え、結果の理由が説明しやすいからです。たとえば「日照が多いほど糖度が上がる」といった大まかな関係を、まず把握できます。

次にランダムフォレストを使います。理由は、現実の栽培は「日照がある値以上で、かつ水分がある値以下のときに糖度が高い」といった、条件が組み合わさった非線形な関係になるからです。ランダムフォレストはこうした関係を捉えられ、さらに「どの要因が効いているか」という特徴量の重要度を出せるので、栽培への示唆に直結します。

---

## 19. 機械学習の手法②：なぜこの手法か（55秒）

データが増えてきたら、次の2つを使います。

一つは LightGBM という勾配ブースティングです。理由は、今回のような表形式のデータに強く、少〜中規模のデータでも高い精度が出やすいうえ、要因どうしの掛け合わせも自動で捉えてくれるからです。学習も速く、試行錯誤しやすい点も向いています。

もう一つは ガウス過程回帰（GPR）です。理由は、糖度や収穫量のように測定回数が限られる、データの少ない予測対象に強いからです。さらに、予測値だけでなく「この予測がどれくらい不確かか」も同時に出せるので、「この条件は十分確かめられている／まだ不確かだから次に試したい」といった栽培の判断にも活かせます。ライブラリは Python の scikit-learn などを使います。

---

## 20. 評価方法：なぜそれで測るか（45秒）

精度の測り方にも理由があります。データは時間の前後で分け、過去で学習して未来を予測する形で評価します。普通のランダムな分け方だと、未来の情報を使って過去を当てる「ズル(リーク)」が起きてしまい、本当の精度が分からないからです。実際の用途は未来予測なので、時間順での評価が欠かせません。ズレの大きさは MAE と RMSE という指標で測ります。糖度や個数のズレを直感的な単位で確認でき、大外れを重く見る RMSE と、平均的なズレを見る MAE の両方で評価します。

---

## 21. データ蓄積とロードマップ（40秒）

当面の一番の課題は、データの量です。機械学習には十分なデータが必要なので、まずは1日数回の測定を続け、糖度や収穫量とあわせてためていきます。天気は1日1回自動で取得・補完されるようにしました。今後は、データ蓄積 → モデルの試作 → 時系列での検証 → 改善、というステップで進めていきます。

---

## 22. まとめ・結び（30秒）

まとめます。前期は、トマトの栽培データを「測る・ためる・見せる」仕組みを、マイコンからサーバー、ブラウザまで一通り作り上げ、通信や写真形式などの課題も乗り越えました。後期は、このデータを機械学習に活かし、最終目標である「糖度と収穫量を高める育て方」をデータから見つけるところまで進めます。データに基づくトマト栽培を目指します。以上で発表を終わります。ありがとうございました。
